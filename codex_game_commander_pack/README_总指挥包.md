# Codex 游戏总指挥包

这是一套给 **Codex + 小团队 / 单人制作** 使用的项目起步包，目标是把你想做的“近现代全球战争 + 角色成长 + 势力经营 + 海陆空作战 + 轻社交”的游戏，拆成 **Codex 能持续执行的仓库结构**。

## 这套包的用途

你不是让 Codex “凭空写一个大作”，而是让它：

1. 先读懂项目目标和禁区。
2. 按里程碑做出可运行的最小版本。
3. 每次只做一个明确功能。
4. 每做完一步就验证、记录、提交。

## 我建议的首发方向

**代号：裂变纪元**

- 类型：近现代战争题材的沙盒策略 RPG / 战役经营游戏
- 视角：2.5D / 俯视角
- 核心：大地图跑图、任务推进、招募人物、加入/背叛/自立、海陆空战役、占领关键节点、历史事件触发
- 社交：先做**轻社交**，避免首发就做重度实时 MMO

## 当前最重要的决策

### 1) 先做 MVP，不做“从鸦片战争到今天”的全史巨作
首发版本只做一个可玩的时间段，推荐：

- 时间：**1894 - 1945**
- 区域：**东亚 + 西太平洋**
- 主要体验：
  - 势力选择
  - 大地图移动
  - 城市/港口/铁路/机场占领
  - 陆战、海战、空袭三种战斗表现
  - 事件链与人物招募
  - 轻社交（排行榜 / 战报 / 军团）

### 2) 先做单机主体，社交做成外挂层
优先顺序：

- P0：完整单机核心循环
- P1：排行榜 / 战报分享 / 玩家档案
- P2：军团 / 阵营协作任务
- P3：异步联机与赛季地图
- P4：实时合作（最后再考虑）

### 3) 先做 Godot 版本
原因：
- 更适合小团队快速迭代
- 文本资源、脚本、场景都更容易让 Codex 读写
- 适合 2D / 2.5D 数据驱动项目
- 成本和工程复杂度更友好

如果后期要冲更高端 3D 表现，再评估 Unity / Unreal。

## 目录说明

- `AGENTS.md`：仓库级指挥原则，Codex 每次进仓库都先看这个
- `PLANS.md`：项目执行计划模板
- `.codex/config.toml`：Codex 的仓库级配置建议
- `.agents/skills/`：给 Codex 的复用技能
- `docs/`：设计、范围、社交、提示词、里程碑
- `game/`：Godot 的最小工程骨架

## 你接下来怎么用

### 方案 A：你自己开仓库 + 用 Codex
1. 新建 Git 仓库
2. 把这整套文件复制进去
3. 用 Codex App / IDE / CLI 打开这个仓库
4. 先让 Codex 阅读：
   - `AGENTS.md`
   - `PLANS.md`
   - `docs/00_GDD_MVP.md`
   - `docs/03_Codex_首批任务提示词.md`
5. 然后给它第一个任务：
   - “请先进入计划模式，阅读 AGENTS.md 与 GDD，只实现第一版世界地图垂直切片，不扩展功能。”

### 方案 B：你让我继续当总指挥
你把 Codex 的输出贴给我，我来继续：
- 审方向
- 拆功能
- 改提示词
- 收口范围
- 控节奏
- 处理“AI 开发跑偏”的问题

## 先做什么最稳

从这 4 件事开始：

1. 世界地图原型  
2. 势力与城市数据结构  
3. 占点 / 补给 / 事件触发  
4. 一场可玩的陆战垂直切片  

等这 4 个稳定了，再扩海战、空战、社交。

## 一个重要原则

**先做“可玩”，再做“庞大”。**
这个项目最大的风险不是技术，而是范围失控。

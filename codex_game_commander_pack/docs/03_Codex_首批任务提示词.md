# 03_Codex_首批任务提示词

下面这些提示词，是给 Codex 的第一批可执行任务。建议一次只发一个。

---

## 提示词 1：初始化仓库理解

请先进入计划模式，完整阅读以下文件：
- AGENTS.md
- PLANS.md
- docs/00_GDD_MVP.md
- docs/01_技术选型.md
- docs/02_社交系统方案.md

你的任务不是立即大规模编码，而是：
1. 复述你对项目的理解；
2. 提炼当前 MVP 范围；
3. 列出最合理的前 6 个开发任务；
4. 给出建议的 Godot 项目目录；
5. 不要实现超出 MVP 的功能。

输出时请明确：
- 当前做什么
- 当前不做什么
- 哪个任务最适合作为第一刀

---

## 提示词 2：建立世界地图垂直切片

基于本仓库文档，为 Godot 项目实现一个最小可运行的世界地图垂直切片。

目标：
- 可以进入主场景
- 显示 8~12 个地图节点
- 节点分为至少 3 类（城市、港口、机场）
- 点击节点可以显示基础信息
- 能切换所属势力显示
- 数据来自配置文件，不要硬编码在 UI 里

限制：
- 不要接入联网
- 不要做复杂美术
- 不要实现完整战斗
- 先把数据结构和交互打通

完成后请输出：
- 新增 / 修改文件
- 运行方式
- 验证步骤
- 后续建议任务

---

## 提示词 3：实现势力与城市数据系统

请实现势力与城市的数据层与加载逻辑。

要求：
- 创建 Faction 和 City 的数据表示
- 支持从 JSON 或 Godot Resource 读取
- 建立城市与势力关系
- 城市至少支持以下字段：
  - id
  - 名称
  - 类型
  - 所属势力
  - 工业值
  - 补给值
  - 战略值
  - 相邻节点
- 势力至少支持以下字段：
  - id
  - 名称
  - 颜色
  - 阵营标签
  - 初始资源

验收标准：
- 主场景可读取并展示这些数据
- 数据修改后界面能反映变化
- 不要把字段散落在多个脚本里

---

## 提示词 4：实现占领与补给最小闭环

请在当前 Godot 项目中实现“占领 + 补给”的最小闭环。

要求：
- 当玩家满足条件后可以占领一个节点
- 被占领节点改变所属势力
- 占领结果影响势力资源或补给值
- 邻接关系影响补给连通
- 给出可见反馈（UI 提示）

限制：
- 不做完整 AI
- 不做复杂路径寻路
- 只做可验证闭环

请同时更新文档。

---

## 提示词 5：做一场最小陆战垂直切片

请实现一场最小的陆战战斗切片。

目标：
- 从世界地图点击某节点后可进入战斗场景
- 玩家控制一个指挥单位或小队
- 至少有移动、射击、胜负结算
- 战斗结果能回写到世界地图
- 战斗胜利后可触发占领

限制：
- 不做复杂动画
- 不做复杂数值平衡
- 不做联网
- 先保证能跑通从世界地图到战斗再返回的闭环

---

## 提示词 6：做首发轻社交原型

请实现首发轻社交的前端和本地数据原型。

范围：
- 玩家档案页
- 勋章展示
- 战报记录
- 排行榜占位页
- 军团入口占位页

限制：
- 先使用本地假数据
- 不做真实联网
- 不做聊天
- 不做实时多人

目标：
- 让项目结构为未来接入在线服务做好准备
- 但当前仍保持单机可运行

---

## 提示词 7：收口和审查

请不要新增功能，而是对当前仓库进行一次审查。

要求：
- 检查是否有超出 MVP 范围的内容
- 检查是否有硬编码数据
- 检查目录结构是否清晰
- 检查文档是否与代码一致
- 列出最值得修的 10 个问题
- 先给审查报告，再等实现

# 04_里程碑与开发节奏

## Sprint 0：立项与骨架（1 周）

目标：
- 项目文档齐
- Godot 空工程可打开
- 世界地图和数据结构方案确定
- Codex 工作流跑通

产出：
- AGENTS.md
- PLANS.md
- GDD
- 项目骨架
- 第一批提示词

## Sprint 1：世界地图 MVP（1~2 周）

目标：
- 地图节点展示
- 节点详情
- 势力切换
- 城市 / 港口 / 机场差异化

## Sprint 2：势力、资源、占领（1~2 周）

目标：
- 势力数据
- 占领逻辑
- 补给连通
- 基础事件触发

## Sprint 3：人物与任务（1~2 周）

目标：
- 招募人物
- 人物属性
- 基础任务链
- 势力关系变化

## Sprint 4：陆战垂直切片（2 周）

目标：
- 地图进入战斗
- 战斗结算
- 回写占领结果

## Sprint 5：海战 / 空战原型（2 周）

目标：
- 海战最小原型
- 空战最小原型
- 接入世界地图入口

## Sprint 6：轻社交原型（1 周）

目标：
- 档案页
- 排行榜
- 战报
- 勋章
- 军团入口占位

## Sprint 7：打磨与收口（1~2 周）

目标：
- 修 bug
- 收范围
- 提升可玩性
- 形成首个可试玩版本

## 原则

- 每个 Sprint 都必须有可运行结果
- 每个 Sprint 都必须有一条“玩家能感知”的提升
- 每个 Sprint 都必须能被录屏验证
- 任何超过 2 周的大任务，必须再拆

# 01_技术选型

## 结论

**首发建议：Godot**
**备选：Unity**
**暂不建议首发直接用 Unreal**

## 为什么首发用 Godot

这个项目第一阶段更像：
- 2.5D / 俯视角
- 数据驱动
- UI 与地图交互很多
- 单机主循环优先
- 小团队 / 单人高频迭代
- 需要让 Codex 高频读写场景、脚本、配置

在这个前提下，Godot 更适合快速落地第一版。

## Godot 项目结构建议

- `game/scenes/`：场景
- `game/scripts/`：脚本
- `game/data/`：配置数据
- `game/assets/`：美术资源
- `game/ui/`：界面资源
- `game/tests/`：验证脚本（后续）

## 技术原则

1. 先数据驱动，再表现
2. 先单机闭环，再联网层
3. 先异步社交，再实时社交
4. 先可用，再复杂

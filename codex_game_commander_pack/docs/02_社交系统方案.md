# 02_社交系统方案

## 社交目标

本项目的社交不是为了把它变成 MMO，而是为了：

- 放大战绩与占领成就感
- 提高留存
- 形成军团归属
- 让玩家有“我参与了时代变化”的感觉

## 推荐的四层社交结构

### 第一层：展示型社交（首发必须有）
这是成本最低、最稳的一层。

功能：
- 玩家档案页
- 历史称号 / 勋章
- 战报分享
- 排行榜
- 赛季记录

优点：
- 实现成本低
- 几乎不破坏单机节奏
- 很适合早期冷启动

### 第二层：弱连接社交（首发可选）
功能：
- 军团 / 同盟
- 军团公告
- 军团目标
- 阵营贡献榜
- 协作任务

优点：
- 有归属感
- 不要求实时在线
- 对独狼玩家也相对友好

### 第三层：异步对抗（中期）
功能：
- 异步占点
- 阵营赛季地图
- 防守编队留驻
- 战报回放
- 结算奖励

优点：
- 成就感强
- 内容消耗慢
- 兼顾单机与竞争

### 第四层：实时协作（后期）
功能：
- 双人合作战役
- 小队联合突袭
- 联合海战
- 实时军团作战

风险：
- 网络复杂度高
- 同步难度高
- 维护成本高
- 很容易拖垮主项目节奏

**建议：后期再做。**

## 首发最优方案

### 必做
- 玩家档案
- 排行榜
- 战报分享
- 勋章系统
- 军团入口占位
- 阵营贡献数值

### 可选
- 军团
- 协作目标
- 军团奖励

### 不做
- 世界频道
- 大规模实时战场
- 实时语音
- 开放世界同地图多人

## 首发社交闭环

1. 玩家完成一场战役
2. 生成战报
3. 战报进入个人档案
4. 战绩进入排行
5. 贡献计入军团 / 阵营
6. 玩家拿到称号或赛季奖励

## 你真正要的社交能力

对于这个项目，最适合的是：

**“异步成就社交 + 阵营归属社交 + 赛季竞争社交”**

而不是一上来就做重度实时在线社交。
